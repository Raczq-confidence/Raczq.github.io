call，apply，bind的区别以及实现

发布日期:   2020-12-29

1. call，apply，bind的区别

首先先来简单地说一下三者的共同点：

  1. 三者都是用来改变函数体内部的this指向
  2. 第一个参数都是this要指向的对象
  3. 后续参数都可以被用来进行传参

以下是MDN文档关于call，apply，bind的语法

call

call() 方法使用一个指定的 this 值和单独给出的一个或多个参数来调用一个函数。

call语法：

fun.call(thisArg, arg1, arg2, ...)
thisArg:：在fun函数运行时指定的this值。需要注意的是，指定的this值并不一定是该函数执行时真正的this值，如果这个函数处于非严格模式下，则指定为null和undefined的this值会自动指向全局对象(浏览器中就是window对象)，同时值为原始值(数字，字符串，布尔值)的this会指向该原始值的自动包装对象。
arg1, arg2, ... 指定的参数列表。

举个例子

var obj = {
	a : 1
}

function test(a,b,c) {

	console.log(this)
	return a + b + c

}

// 下面两个函数调用的返回结果全部为NaN
test() //输出结果为全局对象window
test.call(obj,1,2) //输出结果为obj对象

apply

apply() 方法调用一个具有给定this值的函数，以及以一个数组（或类数组对象）的形式提供的参数。

apply语法：

fun.apply(thisArg, [argsArray])
thisArg： 在 fun 函数运行时指定的 this 值。需要注意的是，指定的 this 值并不一定是该函数执行时真正的 this 值，如果这个函数处于非严格模式下，则指定为 null 或 undefined 时会自动指向全局对象（浏览器中就是window对象），同时值为原始值（数字，字符串，布尔值）的 this 会指向该原始值的自动包装对象。
argsArray: 一个数组或者类数组对象，其中的数组元素将作为单独的参数传给 fun 函数。如果该参数的值为null 或 undefined，则表示不需要传入任何参数。从ECMAScript 5 开始可以使用类数组对象。

举一个与上面的call类似的例子

var obj = {
	a : 1
}

function test(a,b,c) {

	console.log(this)
	return a + b + c

}

// 下面两个函数调用的返回结果全部为NaN
test() //输出结果为全局对象window
test.apply(obj,[1,2]) //输出结果为obj对象

bind

bind() 方法创建一个新的函数，在 bind() 被调用时，这个新函数的 this 被指定为 bind() 的第一个参数，而其余参数将作为新函数的参数，供调用时使用。

bind语法：

func.bind(thisArg[, arg1[, arg2[, ...]]])
thisArg 当绑定函数被调用时，该参数会作为原函数运行时的this指向。当使用new 操作符调用绑定函数时，该参数无效。
arg1, arg2, ... 当绑定函数被调用时，这些参数将置于实参之前传递给被绑定的方法。

bind函数我这里使用了MDN文档上面的bind例子

  1. 创建绑定函数例子
this.x = 9;    // 在浏览器中，this 指向全局的 "window" 对象
var module = {
  x: 81,
  getX: function() { return this.x; }
};

module.getX(); // 81

var retrieveX = module.getX;
retrieveX();
// 返回 9 - 因为函数是在全局作用域中调用的

// 创建一个新函数，把 'this' 绑定到 module 对象
// 新手可能会将全局变量 x 与 module 的属性 x 混淆
var boundGetX = retrieveX.bind(module);
boundGetX(); // 81
  1. 偏函数例子
function list() {
  return Array.prototype.slice.call(arguments);
}

function addArguments(arg1, arg2) {
    return arg1 + arg2
}

var list1 = list(1, 2, 3); // [1, 2, 3]

var result1 = addArguments(1, 2); // 3

// 创建一个函数，它拥有预设参数列表。
var leadingThirtysevenList = list.bind(null, 37);

// 创建一个函数，它拥有预设的第一个参数
var addThirtySeven = addArguments.bind(null, 37);

var list2 = leadingThirtysevenList();
// [37]

var list3 = leadingThirtysevenList(1, 2, 3);
// [37, 1, 2, 3]

var result2 = addThirtySeven(5);
// 37 + 5 = 42

var result3 = addThirtySeven(5, 10);
// 37 + 5 = 42 ，第二个参数被忽略

区别总结：

​ 三者都是用于改变函数体内this指针的指向。

  1. call与apply是为了改变某个函数运行时的上下文而存在的（就是为了改变函数内部this指针的指向）,call与apply调用的返回值为函数的执行结果；
  2. 如果使用apply或call方法，那么this指向他们的第一个参数，apply的第二个参数是一个参数数组，call的第二个及其以后的参数都是数组里面的元素，就是说要全部列举出来；
  3. bind与apply和call的最大的区别是：bind不会立即调用，而是返回一个新函数，称为绑定函数，其内的this指向为创建它时传入bind的第一个参数，而传入bind的第二个及以后的参数作为原函数的参数来调用原函数。

2. 实现方法

call实现方法

function myCall(context) {

	// 基础类型转换
    if(context === null || context === undefined) {
        context = window
    } else if(typeof context === 'string') {
        context = new String(context)
    } else if(typeof context === 'boolean') {
       	context = new Boolean(context) 
    } else if(typeof context === 'number') {
        context = new Number(context)
    }
	
	// 定义一个函数返回结果
	var result
    
    // 保存需要被调用的函数到context对象上
    context.fn = this
    
    // 获取除第一个参数context外的其他参数
    var args = Array.from(arguments).slice(1)
	
	// 将函数执行的返回结果存入result
	result = context.fn(...args)
    
    // 将context上的fn属性清除
    delete context.fn
    
    // 返回值为result
	return result
}

// 将此函数暴露在Function的原型上
Function.prototype.myCall = myCall

apply实现方法

function myApply(context,array) {
    
    // 基础类型转换
    if(context === null || context === undefined) {
        context = window
    } else if(typeof context === 'string') {
        context = new String(context)
    } else if(typeof context === 'boolean') {
       	context = new Boolean(context) 
    } else if(typeof context === 'number') {
        context = new Number(context)
    }
    
    // 注意：当array的类型不为对象，undefined，null时才会报错(当typeof null时，值为'object')
    if(typeof array !== 'object' && typeof array !== 'undefined') {
        throw new Error('第二个参数有问题')
    }
    
    // 判断array是否是数组
    array = Array.isArray(array) ? array : []
    
    // 保存需要被调用的函数到context对象上
    context.fn = this
    
    // 将函数执行的返回结果存入result
    var result = context.fn(...array)
    
    // 将context上的fn属性清除
    delete context.fn
    
    // 返回值为result
    return result
    
}

// 将此函数暴露在Function原型上
Function.prototype.myApply = myApply

bind实现方法

function myBind(context) {
    
    // 基础类型转换
    if(context === null || context === undefined) {
        context = window
    } else if(typeof context === 'string') {
        context = new String(context)
    } else if(typeof context === 'boolean') {
       	context = new Boolean(context) 
    } else if(typeof context === 'number') {
        context = new Number(context)
    }
    
    // 将this指针保存下来
    var pointer = this
    
    // 获取除了第一个参数的所有参数
    var args = Array.from(arguments).slice(1)
    
    /* 
    	定义一个用来判断这个bind函数的返回值是否被new实例化调用的
    	如果是，则使用pointer(即指向自己)
    	如果不是，则使用context
    */
    function isConstructor(){}
    function fn(){
        
        // 第一个参数就是绑定的指针
        // 第二个参数就是将bind和fn的参数合并
        pointer.apply(this instanceof isConstructor ? this : context,args.concat(Array.from(arguments)))
        
    }
	
	// 将上面函数作为bind函数的返回值
	return fn
	
}

// 将此函数挂载在Function的原型上
Function.prototype.myBind = myBind
